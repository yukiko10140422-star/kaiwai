
# KAIWAI - 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
KAIWAI

### 1.2 目的
プロジェクト進行中のチーム内連絡・タスク調整を行うWebアプリケーション。
Notion（ドキュメント・タスク管理）と Slack（チャット・連絡）の機能を統合した、軽量・低コストなツール。

### 1.3 背景
- 既存ツール（Notion, Slack等）はチーム利用時のコストが高い
- 必要な機能に絞った自作ツールでコストを抑える

### 1.4 対象ユーザー
- チームメンバー（最大10人程度）
- 招待制（管理者がメンバーを招待）

## 2. 技術スタック

| カテゴリ | 技術 | 備考 |
|----------|------|------|
| フロントエンド | Next.js (React) | Vercelとの相性最適 |
| スタイリング | Tailwind CSS | モダンUI・高品質デザイン |
| アニメーション | Framer Motion | スムーズなUI遷移・インタラクション |
| データベース | Supabase (PostgreSQL) | 認証・DB・ストレージ統合 |
| 認証 | Supabase Auth | メール+パスワード、OAuth対応 |
| リアルタイム通信 | Supabase Realtime | チャット・通知のリアルタイム配信 |
| ファイルストレージ | Supabase Storage | 画像・動画・ドキュメント保存 |
| ホスティング | Vercel | Next.jsの最適デプロイ先 |
| 言語 | TypeScript | 型安全な開発 |

## 3. 機能要件

### 3.1 認証・ユーザー管理

| 機能 | 説明 | 優先度 |
|------|------|--------|
| サインアップ | メールアドレス+パスワードで新規登録 | 必須 |
| サインイン | メールアドレス+パスワードでログイン | 必須 |
| サインアウト | ログアウト | 必須 |
| プロフィール設定 | 表示名・アイコン画像の設定・変更 | 必須 |
| パスワードリセット | メール経由でパスワード再設定 | 高 |
| 招待機能 | 管理者がメールで招待リンクを送信 | 高 |

### 3.2 連絡ツール（チャット機能） - Slack風

| 機能 | 説明 | 優先度 |
|------|------|--------|
| チャンネル | トピック別のチャットルーム作成・管理 | 必須 |
| リアルタイムメッセージ | チャンネル内でのテキストメッセージ送受信（Supabase Realtime） | 必須 |
| ダイレクトメッセージ | メンバー間の1対1チャット | 必須 |
| メンション | @ユーザー名 で特定メンバーに通知 | 高 |
| ファイル共有 | 画像・動画・ドキュメント等の添付送信 | 必須 |
| メッセージ検索 | 過去のメッセージをキーワード検索 | 中 |
| 未読管理 | 未読メッセージの表示・通知バッジ | 高 |
| スレッド返信 | メッセージへのスレッド形式での返信 | 中 |
| リアクション | メッセージへの絵文字リアクション | 低 |

### 3.3 ファイル共有・資料管理

| 機能 | 説明 | 優先度 |
|------|------|--------|
| 画像アップロード | PNG, JPG, GIF, WebP 等 | 必須 |
| 動画アップロード | MP4, MOV 等（プレビュー再生対応） | 必須 |
| ドキュメント共有 | PDF, Word, Excel, PowerPoint 等 | 必須 |
| その他ファイル | ZIP, CSV 等あらゆるファイル形式 | 高 |
| プレビュー表示 | 画像・動画・PDFのインライン表示 | 高 |
| ドラッグ&ドロップ | ファイルをドラッグで簡単アップロード | 高 |
| ファイル一覧 | チャンネル・DM内の共有ファイル一覧表示 | 中 |
| ストレージ容量管理 | Supabase Storage の容量監視 | 中 |

### 3.4 タスク管理（タスク調整機能） - Notion風

| 機能 | 説明 | 優先度 |
|------|------|--------|
| タスク作成 | タイトル・説明・期限・担当者を設定 | 必須 |
| タスク一覧 | リスト表示で全タスクを確認 | 必須 |
| カンバンボード | ステータス別（未着手/進行中/レビュー/完了）のボード表示 | 必須 |
| 担当者アサイン | タスクにメンバーを割り当て（アイコン表示） | 必須 |
| 期限設定 | タスクの締切日を設定（期限切れ警告表示） | 必須 |
| ステータス変更 | ドラッグ&ドロップでステータス変更 | 必須 |
| 進捗バー | タスク全体の進捗率をビジュアル表示 | 必須 |
| ラベル・タグ | タスクの分類用カラーラベル | 高 |
| コメント | タスクへのコメント・議論 | 高 |
| フィルター・ソート | 担当者・ステータス・期限・ラベルでフィルタリング | 高 |
| プロジェクト分類 | タスクをプロジェクト単位でグループ化 | 高 |
| タイムライン表示 | ガントチャート風の期間表示 | 中 |
| サブタスク | タスク内に子タスクを作成（チェックリスト形式） | 高 |
| 優先度設定 | 高/中/低の優先度アイコン表示 | 高 |

### 3.5 進捗ダッシュボード

| 機能 | 説明 | 優先度 |
|------|------|--------|
| プロジェクト進捗率 | 全タスクの完了率を円グラフ・進捗バーで表示 | 必須 |
| メンバー別進捗 | 各メンバーの担当タスク消化状況 | 高 |
| 期限カレンダー | タスク期限をカレンダー上に表示 | 高 |
| 今週のタスク | 今週期限のタスクをハイライト | 高 |
| アクティビティログ | 誰がいつ何をしたかの履歴表示 | 中 |
| 統計サマリー | 完了数/未完了数/遅延数のサマリーカード | 高 |

### 3.6 通知機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| アプリ内通知 | メンション・タスクアサイン・期限アラート | 必須 |
| 通知一覧 | 通知の一覧・既読管理 | 高 |
| ブラウザ通知 | プッシュ通知（ブラウザ通知API） | 中 |

## 4. デザイン方針

### 4.1 コンセプト
- **モダン & プレミアム**: ダークモード対応、グラスモーフィズム、微細なグラデーション
- **直感的操作**: ドラッグ&ドロップ、スムーズなアニメーション遷移
- **情報の視認性**: 進捗状況が一目でわかるビジュアルデザイン

### 4.2 デザインシステム
- Tailwind CSS ベースのカスタムテーマ
- Framer Motion によるページ遷移・要素アニメーション
- レスポンシブ対応（PC優先、モバイルも使用可能）
- ダークモード / ライトモード切り替え

### 4.3 UI コンポーネント例
- グラスモーフィズムのカード（backdrop-blur + 半透明背景）
- グラデーション付きプログレスバー
- スムーズなドラッグ&ドロップ（カンバンボード）
- アバターアイコン付きメンバーリスト
- フローティング通知トースト
- ホバー時のマイクロインタラクション

## 5. 画面構成

```
1. サインイン / サインアップページ
   └── モダンな認証フォーム（グラスモーフィズム）

2. メインダッシュボード
   ├── サイドバー（折りたたみ可能）
   │   ├── ユーザープロフィール（アイコン・名前・ステータス）
   │   ├── チャンネル一覧
   │   ├── ダイレクトメッセージ一覧
   │   ├── タスクボードへのリンク
   │   └── 設定
   ├── チャットエリア（メイン表示）
   │   ├── チャンネルヘッダー（メンバー・ファイル一覧）
   │   ├── メッセージ一覧（画像・動画プレビュー対応）
   │   └── メッセージ入力欄（ファイルドラッグ&ドロップ）
   └── 右パネル（トグル表示）
       ├── スレッド詳細
       ├── メンバー情報
       └── 共有ファイル一覧

3. タスクボード
   ├── ヘッダー（プロジェクト切り替え・フィルター）
   ├── ビュー切り替え（カンバン / リスト / タイムライン）
   ├── カンバンビュー（ドラッグ&ドロップ列）
   ├── 進捗ダッシュボード（グラフ・統計）
   └── タスク詳細モーダル

4. 進捗ダッシュボード
   ├── プロジェクト全体の進捗率（円グラフ）
   ├── メンバー別タスク状況
   ├── 期限カレンダー
   └── アクティビティログ

5. プロフィール設定ページ
6. 管理者設定ページ（メンバー招待・権限管理）
```

## 6. 非機能要件

| 項目 | 内容 |
|------|------|
| 同時接続数 | 最大10人 |
| リアルタイム | Supabase Realtime（チャット・通知・タスク更新） |
| デバイス対応 | PC ブラウザ優先（レスポンシブでモバイル対応） |
| 認証 | Supabase Auth（JWT、RLS） |
| ファイル保存 | Supabase Storage（画像・動画・ドキュメント） |
| セキュリティ | Row Level Security、HTTPS |
| パフォーマンス | Next.js SSR/SSG、画像最適化（next/image） |

## 7. 開発フェーズ（案）

### Phase 1: 基盤構築
- Next.js + Tailwind CSS プロジェクトセットアップ
- Supabase プロジェクト作成・DB設計
- 認証機能（サインイン/アップ/アウト）
- プロフィール設定（アイコン・名前）

### Phase 2: チャット機能
- チャンネル作成・管理
- リアルタイムメッセージ送受信
- ダイレクトメッセージ
- ファイル共有（画像・動画・ドキュメント）

### Phase 3: タスク管理
- タスクCRUD
- カンバンボード（ドラッグ&ドロップ）
- 担当者アサイン・期限設定
- サブタスク・ラベル

### Phase 4: 進捗ダッシュボード
- プロジェクト進捗率グラフ
- メンバー別進捗
- 期限カレンダー
- アクティビティログ

### Phase 5: 仕上げ・デプロイ
- 通知機能
- 検索機能
- UIアニメーション・デザイン仕上げ
- Vercel デプロイ・本番運用
